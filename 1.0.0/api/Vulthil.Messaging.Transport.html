Namespace Vulthil.Messaging.Transport

Classes

ConsumePipelineFactory

Composes the registered IConsumeFilter<TMessage> instances into a single delegate. Transport authors call this in their consume path to wrap a terminal stage (the consumer invocation) with the configured middleware.

HandlerEntry<THandler>

Pairs a transport-specific dispatch handler with the consumer contract it implements. Returned by IMessageHandlerFactory<THandler> so MessageExecutionRegistry<THandler> can enforce request-consumer uniqueness without inspecting the opaque THandler value.

MessageContext

Transport-agnostic IMessageContext implementation describing a delivered message and the transport services that back its PublishAsync<TMessage>(TMessage, Func<IPublishContext, ValueTask>?) and SendAsync<TMessage>(Uri, TMessage, Func<IPublishContext, ValueTask>?) helpers. A transport builds one per delivery (typically via CreateFromEnvelope<TMessage>(TMessage, MessageEnvelope, string, bool, int, string?, IPublisher?, ISendEndpointProvider?, CancellationToken)); a context with no live transport binding (e.g. a fault snapshot) throws when a publish or send is attempted.

MessageContext<TMessage>

A strongly-typed MessageContext carrying the deserialized message payload.

MessageEnvelope

On-the-wire wrapper for messages carried by a Vulthil transport. The body of every produced message is a serialized MessageEnvelope; receivers fall back to bare JSON only for compatibility with non-Vulthil producers.

MessageEnvelopeFactory

Builds MessageEnvelope instances from the resolved publish state for a single outgoing message. Promotes the reserved metadata headers carried by a PublishContext to typed envelope fields and copies the remaining custom headers verbatim.

MessageExecutionPlan<THandler>

Transport-agnostic execution plan for a single concrete message type: the ordered set of handlers that run on every delivery, plus the partition specification when the type is partitioned for ordered consumption. Built and keyed by MessageExecutionRegistry<THandler>.

MessageExecutionRegistry<THandler>

Transport-agnostic registry that turns QueueDefinition registrations into per-message-type MessageExecutionPlan<THandler> instances. It keys plans by wire URN (and CLR full name for the bare-JSON receive path), fans a polymorphic registration out across every matching concrete subscription, dedupes handlers, rejects a second request consumer for the same message type, and attaches the partition specification read from GetPartition(Type). A transport drives it by supplying an IMessageHandlerFactory<THandler> that builds its own delivery closures.

PublishContext

Mutable, write-through configuration captured for a single outgoing message. A transport creates one, passes it to the caller's configure callback as IPublishContext, then reads the resolved values to build its wire message. Metadata that maps to typed envelope fields is stored under reserved header keys and surfaced through the strongly-typed properties.

PublishFilterContext

The context flowing through the publish/send filter pipeline for a single outgoing message. A filter may inspect the message and its resolved metadata, adjust Context, short-circuit the pipeline by not invoking next (e.g. to capture the message into a transactional outbox), or let it proceed to the transport terminal.

PublishFilteringServiceCollectionExtensions

Registers the publish/send filtering facade in front of a transport's raw publish terminal. A transport registers its publisher and send-endpoint provider under ITransportPublisher / ITransportSendEndpointProvider and then calls AddPublishFiltering(IServiceCollection) so the public IPublisher / ISendEndpointProvider run every publish and send through the pipeline.

PublishPipelineFactory

Composes the registered IPublishFilter instances around a terminal delegate (the transport send). The filtering publisher/send-endpoint call this in their publish path to wrap the transport with the configured middleware.

RequestContext

A PublishContext for request/reply: adds the per-request timeout. A transport creates one, passes it to the caller's configure callback as IRequestContext, then reads Timeout to bound the wait for a response.

RpcFault

Wire payload for an RPC failure reply. Carried as the message of a MessageEnvelope whose messageType is UrnUri. Property names are fixed (camelCase) so the reply round-trips regardless of the configured JSON naming policy.

Interfaces

IMessageHandlerFactory<THandler>

Builds transport-specific dispatch handlers from the open-generic consumer/message type pairs collected during registration. A transport implements this to bind a registration to its own delivery closure; MessageExecutionRegistry<THandler> calls it while assembling execution plans.

IPublishFilter

A filter in the publish/send pipeline. Filters wrap the transport send, allowing cross-cutting concerns (transactional outbox capture, logging, validation, telemetry, …) to be composed without modifying transport or caller code.

ITransportPublisher

A transport's raw publish terminal: it builds the wire envelope and sends it to the broker. The public IPublisher registered to callers is a filtering facade that runs the publish pipeline and then delegates to this terminal. Transports register their publisher under this interface instead of IPublisher.

ITransportSendEndpointProvider

A transport's raw send-endpoint provider. The public ISendEndpointProvider registered to callers is a filtering facade that wraps the returned ISendEndpoint with the publish pipeline. Transports register their provider under this interface instead of ISendEndpointProvider.
