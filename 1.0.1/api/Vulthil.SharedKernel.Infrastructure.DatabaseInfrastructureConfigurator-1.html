Fluent configurator for database infrastructure including outbox processing. The DbContext itself is registered by provider-specific extensions (e.g. UseNpgsql) via the OnConfigured(Action<IDatabaseInfrastructureConfigurator<TDbContext>>) hook, not by this class.

OnConfigured(Action<IDatabaseInfrastructureConfigurator<TDbContext>>)

Registers a callback that runs once the configurator action has finished executing but before the DbContext is registered with the service collection. Use this hook from provider-specific extension methods (e.g. UseNpgsql) to defer work that depends on the final state of the configurator — most notably whether outbox processing has been enabled — regardless of the order in which the user chained calls.
