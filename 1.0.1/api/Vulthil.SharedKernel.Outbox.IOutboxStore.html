Persistence-agnostic store the outbox engine relies on for both capture and relay. A relational or other EF Core implementation lives in Vulthil.SharedKernel.Outbox.EntityFrameworkCore and its provider packages; the engine itself takes no dependency on EF Core.

Remarks

The store owns the whole relay batch unit — provider locking, the transactional boundary, fetching the pending rows, dispatching each, recording success/failure, and committing — so a relational implementation can run it inside an EF Core execution strategy. The capture members are used by message-bridge publish filters to enlist an outgoing message in the ambient business transaction.
