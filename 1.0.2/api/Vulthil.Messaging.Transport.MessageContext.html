Class MessageContext

Namespace
Vulthil.Messaging.Transport
Assembly
Vulthil.Messaging.dll

Transport-agnostic IMessageContext implementation describing a delivered message and the transport services that back its PublishAsync<TMessage>(TMessage, Func<IPublishContext, ValueTask>?) and SendAsync<TMessage>(Uri, TMessage, Func<IPublishContext, ValueTask>?) helpers. A transport builds one per delivery (typically via CreateFromEnvelope<TMessage>(TMessage, MessageEnvelope, string, bool, int, string?, IPublisher?, ISendEndpointProvider?, CancellationToken)); a context with no live transport binding (e.g. a fault snapshot) throws when a publish or send is attempted.

CreateFromEnvelope<TMessage>(TMessage, MessageEnvelope, string, bool, int, string?, IPublisher?, ISendEndpointProvider?, CancellationToken)

Builds a live typed context from a parsed MessageEnvelope. Transport-level fields the envelope does not carry are supplied by the caller: the broker routingKey, the redelivered flag, the in-memory retryCount, and a replyToFallback used for the response address when the envelope omits one.

public static MessageContext<TMessage> CreateFromEnvelope<TMessage>(TMessage message, MessageEnvelope envelope, string routingKey, bool redelivered, int retryCount, string? replyToFallback, IPublisher? publisher, ISendEndpointProvider? sendEndpointProvider, CancellationToken cancellationToken)

PublishAsync<TMessage>(TMessage, Func<IPublishContext, ValueTask>?)

Publishes a message via the underlying transport, automatically propagating correlation metadata (CorrelationId, ConversationId, InitiatorId) from the incoming context to the outgoing message. The caller-supplied configure callback runs after propagation, so it can override any auto-set value.

SendAsync<TMessage>(Uri, TMessage, Func<IPublishContext, ValueTask>?)

Sends a message point-to-point to the specified destination, automatically propagating correlation metadata (CorrelationId, ConversationId, InitiatorId) from the incoming context to the outgoing message.
