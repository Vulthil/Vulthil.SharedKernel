Class CosmosTestContainerFixture<TDbContext>

Namespace
Vulthil.xUnit.Cosmos

Fixture that starts the Azure Cosmos DB Linux emulator in a container and registers TDbContext in the test host pointed at it. Connectivity goes through the container's own Testcontainers.CosmosDb.CosmosDbContainer.HttpClient, whose handler rewrites every request to the mapped emulator endpoint, so no certificate or endpoint plumbing is needed. When owned by a ContainerHost, every consuming factory works in its own emulator database, so parallel test classes never share data; data is reset between tests by recreating that database. Derived classes only supply the configuration key (and optionally pin the emulator image via Configure).

Remarks

The database is created and reset through the application's own TDbContext resolved from the test host's service provider — so a context whose constructor takes more than its options works without extra plumbing. It is created once during host startup through InitializeAsync(IServiceProvider) and recreated after each test through ResetAsync(IServiceProvider). Only the emulator readiness probe and the best-effort scope teardown use a bare DbContext, because neither needs the model.
