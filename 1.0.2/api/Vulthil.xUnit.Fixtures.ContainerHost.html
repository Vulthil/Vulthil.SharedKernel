Remarks

Register a derived host with [assembly: AssemblyFixture(typeof(MyContainerHost))] and accept it in the constructor of a BaseWebApplicationFactory<TEntryPoint>-derived class fixture. The factory consumes every container registered here through a per-factory scope view (see ITestContainerScopeProvider), so parallel test classes share the running containers without sharing state.

Containers start lazily on first use and concurrent consumers share a single startup task, so a filtered test run only pays for the containers its factories actually consume. Assembly fixtures are used from many tests simultaneously; this host is safe for that concurrency.
