Remarks

Values are normalized when the context is built, so every consume path (envelope, bare AMQP, outbox relay) surfaces the same CLR primitives: strings arrive as string, booleans as bool, and numbers as int, long, or double — the narrowest of the three that represents the value. Header values published as non-primitive objects have no CLR type on the wire and surface in their JSON form: as a JsonElement when published as an object or array, or as a string for types JSON represents as strings (e.g. Guid, DateTimeOffset).
