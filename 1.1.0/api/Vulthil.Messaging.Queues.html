Interfaces

IBaseConfigurator<TConfigurator>

Self-typed base interface for fluent consumer configurators. TConfigurator is the derived configurator interface (e.g. IConsumerConfigurator<TConsumer>), so UseRetry(Action<RetryPolicyConfigurator>) returns it directly — concrete configurator classes inherit Vulthil.Messaging.Queues.BaseConfigurator<TConfigurator> without needing their own explicit interface implementations.

IConsumerConfigurator<TConsumer>

Configures per-consumer settings. Binding patterns live on Subscribe<TMessage>(string?); routing-key formatters for the producer side live on MessageConfiguration<TMessage>.

IQueueConfigurator

Configures a queue's consumers, retry policies, and dead letter settings.

IRequestConfigurator<TConsumer>

Configures per-request-consumer settings. Binding patterns live on Subscribe<TMessage>(string?); routing-key formatters for the producer side live on MessageConfiguration<TMessage>. Configuring UseRetry(Action<RetryPolicyConfigurator>) here throws at configuration time: request consumers do not retry — a thrown exception is returned to the requester as an RPC fault reply.
