Namespace Vulthil.Messaging

Classes

ConsumeFilterOptions

Configures the built-in consume filters that AddMessaging registers by default. Each flag toggles a specific filter on or off; a filter whose flag is false is never registered in DI, so deliveries do not pass through it at all.

DependencyInjection

Provides extension methods for registering messaging services.

MessageConfiguration

Configuration used when publishing or binding a message type. Contains exchange declaration settings, the stable wire URN, and optional formatters for routing keys and correlation ids.

MessageConfiguration<TMessage>

Configuration options for a specific message type, including exchange, routing key and correlation ID selection.

PartitionSpec

Records how a message type is partitioned for ordered consumption: the Partitioner whose lanes serialize same-key processing, and the typed key selector (held as a Delegate because the message type is only known generically at registration). The transport resolves the concrete selector via the registered message type.

Partitioner

Distributes work across a fixed number of ordered partitions ("lanes"), selected by a hash of a partition key. Work submitted for the same key runs strictly sequentially and in submission order; work for keys that hash to different lanes runs concurrently.

Interfaces

IMessageConfigurationProvider

Read-only view of the messaging options resolved at runtime by transports, consumers, and filters. Backed by the same instance that IMessagingOptionsConfigurator writes to during composition.

IMessagingConfigurator

Fluent configurator for registering messaging queues, consumers, and options.

IMessagingOptionsConfigurator

Mutable view of the messaging options used during composition. Passed to the action of ConfigureMessagingOptions(Action<IMessagingOptionsConfigurator>).

ITransport

Represents the message transport responsible for starting consumer connections.
