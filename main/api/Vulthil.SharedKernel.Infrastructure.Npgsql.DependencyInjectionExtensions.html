Remarks

The actual call to AddNpgsqlDbContext is deferred until the full configurator chain has executed, so the order of UseNpgsql<TDbContext>(IDatabaseInfrastructureConfigurator<TDbContext>, string, Action<NpgsqlEntityFrameworkCorePostgreSQLSettings>?, Action<DbContextOptionsBuilder>?), EnableOutboxProcessing(Action<OutboxProcessingOptions>?), and UseOutboxStore<TStore>() is irrelevant: the PostgreSQL-optimized outbox store is applied as a default, so a custom store selected via UseOutboxStore is preserved no matter where in the chain this method is called. The outbox processor runs its transactional unit inside the context's execution strategy (Database.CreateExecutionStrategy().ExecuteAsync), so a retrying execution strategy is fully supported and there is no need to force DisableRetry. All settings — including CommandTimeout — are left to the caller.
