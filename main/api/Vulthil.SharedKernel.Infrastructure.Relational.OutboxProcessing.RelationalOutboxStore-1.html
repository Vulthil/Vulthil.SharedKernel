Outbox store for relational database providers. Records processed and failed messages with set-based ExecuteUpdate calls instead of materializing rows. Provider packages (e.g. Npgsql, MySQL) inherit from this class and override FetchMessagesAsync(int, int, CancellationToken) to add row-level locking.

DeleteProcessedAsync(DateTimeOffset, int, CancellationToken)

Deletes up to batchSize outbox rows that were processed or dead-lettered before olderThanUtc. Pending rows are never deleted.

public override Task<int> DeleteProcessedAsync(DateTimeOffset olderThanUtc, int batchSize, CancellationToken cancellationToken)

Parameters

olderThanUtc DateTimeOffset

The cutoff; only terminal rows whose ProcessedOnUtc or FailedOnUtc is older than this are deleted.

batchSize int

The maximum number of rows to delete in this call.

cancellationToken CancellationToken

A token to observe for cancellation.

Returns

Task<int>

The number of rows deleted.

Remarks

Selects the keys of the oldest eligible rows first and deletes by key, so a single call never removes more than batchSize rows — a large backlog is drained in bounded, short-lived deletes instead of one long-locking statement.

UpdateMessagesAsync(IReadOnlyList<Guid>, IReadOnlyList<OutboxMessageFailure>, int, DateTimeOffset, CancellationToken)

Records processed and failed messages, dead-lettering any that reach maxRetries. The default fetches every affected row in a single query, applies the outcome in memory, and saves once; relational providers override this with set-based ExecuteUpdate calls.
