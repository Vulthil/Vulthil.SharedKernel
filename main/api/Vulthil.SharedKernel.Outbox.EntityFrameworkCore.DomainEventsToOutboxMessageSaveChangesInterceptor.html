SavedChangesAsync(SaveChangesCompletedEventData, int, CancellationToken)

Wakes the outbox relay after a save that committed outside an explicit transaction, so domain events captured by a bare SaveChanges are relayed promptly instead of waiting for the next poll. When a transaction is open the relay is signalled on commit by the transaction-commit interceptor instead, so this skips that case to avoid waking the relay before the rows are committed and visible.
