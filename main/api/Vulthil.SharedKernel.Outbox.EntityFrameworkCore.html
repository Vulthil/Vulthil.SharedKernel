DomainEventsToOutboxMessageSaveChangesInterceptor

EF Core save-changes interceptor that captures domain events from tracked aggregate roots and persists them as OutboxMessage entries before the main save completes.

EntityFrameworkOutboxStore<TContext>

Entity Framework Core implementation of IOutboxStore. It owns the relay batch unit — runs inside the context's execution strategy, opens the transaction, fetches a batch, dispatches each message, records the outcome, and commits — and exposes the capture surface used by bus-publish filters. Provider packages derive from this type to add row-level locking (FetchMessagesAsync(int, int, CancellationToken)), set-based updates (UpdateMessagesAsync(IReadOnlyList<Guid>, IReadOnlyList<OutboxMessageFailure>, int, DateTimeOffset, CancellationToken)), or a no-op transaction (BeginTransactionAsync(CancellationToken)).

OutboxModelBuilderExtensions

ModelBuilder extensions that apply the provider-agnostic OutboxMessage mapping.

Interfaces

IOutboxInterceptor

Marker for EF Core interceptors that the outbox attaches to an outbox-enabled DbContext. Implementations are resolved from DI and added to the context options when outbox processing is enabled, so a provider package (e.g. relational) can contribute a transaction-commit interceptor without the provider-agnostic base needing to reference relational-only interceptor types.

ISaveOutboxMessages

Exposes the OutboxMessage entity set for persistence contexts that support outbox integration.
